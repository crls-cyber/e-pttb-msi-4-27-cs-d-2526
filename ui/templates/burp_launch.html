⚠️ Why Burp Suite is not directly integrated

Burp Suite Community Edition does not expose a free REST API for automation. The Pro/Enterprise versions do, but are commercial products. Integration via the free version requires manual operation and XML export.

  • Burp Suite Community has no programmable API (unlike ZAP which has a free REST API)
  • Burp requires a GUI to operate — cannot run headless in a Docker container
  • Burp Pro API exists but requires a paid license

Solution: Run Burp Suite manually on Kali, perform your tests, then export the findings as XML and import them into the Toolbox using the upload feature below.

🛠️ How to export Burp Suite findings to XML

Step 1 — Launch Burp Suite on Kali

# Launch Burp Suite Community from terminal
burpsuite &
# Or from the Kali menu:
# Applications → Web Application Analysis → burpsuite

Step 2 — Configure your browser proxy

In Firefox:

  1. Settings → Network Settings → Manual proxy configuration
  2. HTTP Proxy: 127.0.0.1 Port: 8080
  3. Check "Also use this proxy for HTTPS"
  4. Install Burp CA certificate: browse to http://burpsuite → CA Certificate

Step 3 — Run your tests

Spider / Crawl:

  1. Target tab → right-click on target → "Scan" or "Spider this host"
  2. Wait for spider to complete

Active Scan (Community — limited):

  1. Right-click on any request → "Do active scan"
  2. Check Scanner tab for findings

Step 4 — Export findings as XML

  1. Go to Target tab → Site map
  2. Right-click on your target domain
  3. Select "Issues" → "Report issues for this host"
  4. Format: select XML
  5. Save to /tmp/burp_report.xml
💡 Alternative: Target → right-click → "Save selected items" → XML format

Step 5 — Expected XML format

<?xml version="1.0"?>
<issues burpVersion="2023.x" exportTime="...">
<issue>
<name>SQL injection</name>
<host ip="192.168.x.x">http://target</host>
<severity>High</severity>
<confidence>Certain</confidence>
<issueDetail>...</issueDetail>
</issue>
</issues>
✅ Accepted file format:
  • .xml — Burp Suite XML export (root element must be <issues>)
  • Maximum file size: 50 MB
  • Auto-detected by the upload page (no need to select parser manually)

📤 Ready to import your Burp Suite findings?

Upload your .xml export to automatically create findings in the Toolbox

📤 Go to Upload Page

Select "Burp Suite" as parser type — or use Auto-detect (.xml files are detected automatically)

📋 Requirements
  • Burp Suite installed on Kali:
    which burpsuite
  • Firefox proxy configured (127.0.0.1:8080)
  • Burp CA certificate installed in browser
  • Target accessible from Kali
🔍 What Burp Detects
  • SQL injection
  • Cross-site scripting (XSS)
  • CSRF vulnerabilities
  • Path traversal
  • Insecure headers
  • SSL/TLS issues
  • Sensitive data exposure
  • Authentication flaws
