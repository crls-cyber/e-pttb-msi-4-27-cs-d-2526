Compliance

// This page is a reminder for the audit team — summarizing how this toolbox was built to meet the security and compliance requirements of the project specification (Cahier des charges — Projet M1 CS 2025). Not a public-facing legal notice: this is a professional tool for a dedicated mission machine, not a website exposed to the internet.
🔑 Authentication & RBAC

Flask-Login session management, hashed passwords (Werkzeug).

3 roles: admin (full control + user/scope management), analyst (launch scans, see own results), viewer (read-only).

Forced password change on first login for new accounts.

🎯 Scope Enforcement

Zero trust by default — no scan launches on a target not explicitly authorized.

Full details: Authorization & Scope →

📋 Audit Logging

Sensitive actions are logged: login, target/user create-delete, password changes, workflow/pivot chain launches.

Each entry records who, what, and when — supporting accountability.

🔐 Data Protection

Password hashing, restricted DB access.

Full details: Encryption →

🇪🇺 GDPR Principles

Data stays on the dedicated machine — no cloud, no external server by default.

Full details: RGPD Compliance →

🏗️ Specification Alignment

Built per the original spec: Python/Flask/Poetry, PostgreSQL, MinIO, Celery/Redis orchestration, Docker Compose, modular plugin system.

13 tools integrated: Nmap, Nuclei, ZAP, SQLmap, Hydra, Subfinder, theHarvester, WhatWeb, Metasploit, Aircrack-ng, Ettercap, Wireshark, Burp Suite.

📖 Known Limitations (Documented, Not Hidden)
  • No HTTPS in development (acceptable for local lab use, required for production deployment).
  • Optional external integrations (VirusTotal, Shodan) require internet access — a deliberate tension with the "data stays on the machine" principle. See Integrations.
  • Single-machine deployment — no built-in multi-mission isolation yet (each machine = one mission, by design, see FUTURE_IMPROVEMENTS.md).
  • UI available in EN/FR toggle, but full French translation is not yet complete.
