// Reminder for the audit team — what is encrypted today in this toolbox, and what should be encrypted at the machine level for a real engagement.
🔐 Encryption in This Toolbox

// Password Storage

User passwords are hashed (never stored in clear text) using Werkzeug's secure password hashing, before being saved to the database.

// Database

PostgreSQL stores all jobs, findings, targets, and audit logs. Database access is restricted to the application's own credentials, not exposed externally by default.

// Full Disk Encryption (Machine Level)

Per the "one dedicated machine per mission" policy, the underlying machine disk should be fully encrypted (e.g. LUKS on Linux) — this protects data at rest in case of physical theft. This is a deployment recommendation, not something the toolbox application itself enforces.

// Known Limitation

Full disk encryption protects against physical theft when the machine is powered off — not against an active compromise while the machine is in use (e.g. keylogger, live exfiltration). Defense in depth (restricted sudoers, audit logs, network isolation) complements encryption rather than replacing it. See Compliance for the full picture.
