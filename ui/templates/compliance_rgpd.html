🔒 How This Toolbox Respects GDPR Principles

// Data Minimization & Locality

All scan results, findings, and reports stay on this machine. No data is sent to a cloud service, external server, or third party by default. The only way data leaves the machine is a manual, deliberate export (CSV/PDF/HTML report) to an isolated network with no internet access, as defined by the "one dedicated machine per mission" policy.

// Right to Erasure

Deleting a Target, a Job, or a User account removes the associated records from the database (cascade delete on Findings/Artifacts tied to a Job). Full mission data wipe is achieved by destroying the machine's encrypted disk at the end of an engagement.

// Access Control

Only authenticated, authorized team members (admin/analyst/viewer roles) can access scan data. See the Authorized page for scope enforcement details.

// Audit Trail

Every sensitive action (login, target creation/deletion, user management, password changes, workflow launches) is logged in the audit_logs table — supporting accountability and traceability requirements.

// Known Limitation

Optional external integrations (VirusTotal, Shodan — see Integrations) require internet access and send data to third-party APIs. These are disabled by default and must be explicitly enabled — a deliberate tension with the "no data leaves the machine" principle, documented honestly rather than hidden.
