🛡️ VirusTotal

Would enrich findings with reputation data: is a discovered IP or domain already known as malicious? Has a file hash been flagged by other AV engines? Useful for cross-referencing targets and artifacts (e.g. uploaded PCAP/Metasploit logs) against a global threat intelligence database.

Status: disabled — button shown but inactive.

🔍 Shodan

Would cross-reference scanned IPs against Shodan's internet-wide scan database — useful for passive reconnaissance (what's exposed publicly about a target before actively touching it) and for validating Nmap findings against an independent external source.

Status: disabled — button shown but inactive.

🤔 Why This Was Left Out of the MVP

// API Keys & Secrets Management

Both services require a personal/team API key — a secret that would need secure storage (encrypted, not in plain config) and per-user management, adding complexity beyond the current scope.

// Internet Dependency

This is the core tension: these are cloud APIs, requiring outbound internet access from the mission machine. That directly conflicts with this toolbox's "data stays on the machine, no cloud" design principle (see RGPD Compliance).

// Rate Limits & Reliability

Free-tier API quotas are limited (VirusTotal: a few requests/minute). Building a reliable feature on top of a rate-limited third-party service adds operational fragility not worth the complexity for a first version.

🌐 A Note on Real-World Practice

This toolbox's "host-only, fully isolated" model is a deliberate simplification for this academic lab — and a realistic choice for highly sensitive engagements (banking, defense, critical infrastructure) where contracts may require air-gapped testing.

In everyday practice, however, a pentester's machine is usually connected to the client's network (required to scan anything) and often has controlled internet access — via a VPN back to their own company infrastructure, for tool updates, or to query services like VirusTotal and Shodan exactly as described above. Isolation in the field is typically enforced by the client segmenting the test network, not by the pentester cutting off internet entirely.
