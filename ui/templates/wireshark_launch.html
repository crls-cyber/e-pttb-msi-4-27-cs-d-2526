⚠️ Why Wireshark is not directly integrated

Wireshark/tshark requires direct access to a physical network interface of the host machine to capture live traffic. Running inside a Docker container with network isolation prevents it from capturing traffic between other hosts.

  • Live packet capture requires raw socket access at kernel level
  • Docker containers are isolated in their own network namespace
  • The Docker Worker can only see its own container traffic, not host network traffic

Solution: Capture traffic directly on Kali using tcpdump or Wireshark, then import the .pcap file into the Toolbox using the upload feature below. The Toolbox will automatically parse the file and create findings.

🛠️ How to capture traffic on Kali

Step 1 — One-time setup: allow tcpdump without sudo

# Grant tcpdump the capability to capture packets without sudo
# Only needs to be done once (persists until tcpdump is updated)
sudo setcap cap_net_raw+eip $(which tcpdump)
✅ After this, tcpdump runs without sudo — professional and secure approach. If tcpdump is updated by apt upgrade, run this command again.

Step 2 — Identify your network interface

# List all interfaces
ip addr show
# Common interfaces in this lab:
# eth0 → Host-Only (192.168.145.x) → WebSRV, WS22
# eth1 → NAT (192.168.200.x) → Metasploitable2

Step 3 — Capture traffic (choose your scenario)

📡 General capture (all traffic on interface):

# Capture all traffic on eth1 (NAT network)
tcpdump -i eth1 -w /tmp/capture.pcap
# Press Ctrl+C to stop

🎯 Targeted capture (specific host and protocol):

# Capture FTP traffic to/from Metasploitable2
tcpdump -i eth1 host 192.168.200.133 and port 21 -w /tmp/ftp_capture.pcap
# Capture HTTP traffic
tcpdump -i eth1 host 192.168.200.133 and port 80 -w /tmp/http_capture.pcap
# Capture a limited number of packets
tcpdump -i eth1 -c 100 -w /tmp/capture.pcap

🔍 Capture with Wireshark GUI:

# Launch Wireshark GUI (if installed)
wireshark &
# Select interface → Start capture → Stop → File → Save as .pcap

Step 4 — What the Toolbox detects in your PCAP

🔴 HIGH severity
  • FTP credentials in clear text
  • Telnet passwords
  • HTTP Basic Auth
🟠 MEDIUM severity
  • Unencrypted HTTP traffic
  • Sensitive URLs in clear text
  • Unencrypted protocols
✅ Accepted file formats:
  • .pcap — Standard Wireshark/tcpdump format
  • .pcapng — Next-generation PCAP format
  • .cap — Alternative capture format
  • Maximum file size: 100 MB
